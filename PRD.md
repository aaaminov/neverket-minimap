# PRD: Neverket Minimap

## 1. Описание

Клиентский мод для Minecraft Java Edition 26.2 на Fabric, добавляющий мини-карту.

Главное отличие от обычных мини-карт: территория отображается только после того, как игрок исследовал её с помощью обычной карты Minecraft.

Мод не должен самостоятельно сканировать или раскрывать окружающий мир.

## 2. Цель

Сделать удобную мини-карту, которая:

* сохраняет ценность ванильных карт;
* не показывает неисследованные территории;
* позволяет использовать созданные карты без постоянного удерживания их в руке;
* подходит для ванильного прохождения и серверов без читерских мини-карт.

## 3. Основной сценарий

1. Игрок создаёт пустую карту.
2. Активирует её в определённой местности.
3. Исследует территорию, заполняя обычную карту.
4. Исследованные пиксели автоматически появляются на мини-карте.
5. Неисследованные области остаются пустыми или затемнёнными.
6. После исследования информация сохраняется и доступна без карты в руке.

## 4. Функциональные требования

### 4.1. Мини-карта

Мини-карта отображается в одном из углов экрана.

На ней показываются:

* исследованная местность;
* позиция игрока;
* направление взгляда;
* стороны света;
* границы доступных данных.

Размер, положение и масштаб мини-карты настраиваются.

### 4.2. Источник данных

Мод использует только данные обычных заполненных карт Minecraft.

Территория добавляется на мини-карту, когда:

* игрок держит активную карту и исследует местность;
* карта обновляется в инвентаре игрока;
* игрок получает уже заполненную карту.

Неисследованные пиксели ванильной карты не должны отображаться как открытая территория.

### 4.3. Объединение карт

Если игрок исследовал несколько соседних карт, мод объединяет их в единую мини-карту.

Необходимо учитывать:

* измерение;
* масштаб ванильной карты;
* координаты центра;
* идентификатор карты;
* исследованные пиксели.

Карты разных измерений отображаются отдельно.

### 4.4. Хранение данных

Данные сохраняются отдельно для каждого мира или сервера.

Рекомендуемое разделение:

* одиночная игра — по идентификатору сохранения;
* сервер — по адресу сервера;
* измерение — отдельный слой данных.

После перезапуска игры исследованные области должны сохраняться.

### 4.5. Управление

Минимальные горячие клавиши:

* показать или скрыть мини-карту;
* изменить масштаб;
* открыть полноэкранную карту;
* открыть настройки.

Клавиши должны переназначаться через стандартное меню Minecraft.

### 4.6. Полноэкранная карта

Отдельный экран показывает все объединённые исследованные карты.

Возможности MVP:

* перемещение карты мышью;
* изменение масштаба;
* отображение позиции игрока;
* переключение между измерениями.

## 5. Настройки

Пользователь может изменить:

* положение мини-карты;
* размер;
* квадратную или круглую форму;
* прозрачность;
* масштаб;
* отображение координат;
* отображение сторон света;
* способ отображения неизвестной территории;
* включение полноэкранной карты.

## 6. Ограничения MVP

В первую версию не входят:

* сущности и другие игроки;
* отображение мобов;
* пещерный режим;
* автоматическое исследование чанков;
* точки смерти;
* пользовательские метки;
* синхронизация данных между игроками;
* данные карт, которых никогда не было у игрока.

## 7. Технические требования

* Minecraft Java Edition 26.2.
* Fabric Loader.
* Fabric API.
* Mojang mappings.
* Клиентская часть не должна требовать установки мода на сервер.
* Рендеринг через API и внутреннюю систему рендеринга Minecraft.
* Минимальное использование mixin; предпочтительно применять Fabric events и публичные игровые API.
* Данные карты не должны отправляться на сторонние серверы.

## 8. Предлагаемые компоненты

### MapDataCollector

Получает данные ванильных карт, доступных игроку, и отслеживает их обновление.

### MapStorage

Хранит пиксели, координаты, масштаб и измерение каждой карты.

### MapAtlas

Объединяет отдельные карты в единое координатное пространство.

### MinimapRenderer

Отрисовывает мини-карту в HUD.

### FullscreenMapScreen

Отображает полноэкранную карту.

### ModConfig

Хранит пользовательские настройки и горячие клавиши.

## 9. Критерии готовности MVP

MVP считается готовым, если:

1. После создания и исследования ванильной карты территория появляется на мини-карте.
2. Неисследованные области не раскрываются.
3. Данные нескольких соседних карт корректно объединяются.
4. Карты Верхнего мира, Незера и Энда не смешиваются.
5. Данные сохраняются после выхода из мира.
6. Мод работает на обычном Fabric-сервере без серверной установки.
7. Мини-карту можно скрыть и изменить её масштаб.
8. Полноэкранная карта показывает всю исследованную территорию.
9. Использование мини-карты не вызывает заметных просадок FPS.

## 10. Возможности после MVP

* пользовательские метки;
* точки смерти;
* названия областей;
* импорт карт из рамок и шалкеровых ящиков;
* совместное исследование через серверный мод;
* поддержка карт разных масштабов в одном атласе;
* экспорт исследованной карты в изображение.
